# Maintainer Docs

This directory is for tracked maintainer-only guidance that still has ongoing value after ACM onboarding.

- `LOGGING_STANDARDS.md` documents the current shared logging expectations for frontend, backend, and Python sidecars.
- `RELEASE_NOTES_TEMPLATE.md` is the manual release-notes scaffold when maintainers want a consistent published format.
- `scripts/release/*.mjs` contains the tracked release-prep, chart-sync, and release-notes helpers that were kept because they are still operationally useful outside ACM.

These docs are not part of the ACM control plane. Treat them as human reference material, not machine-enforced contract.
