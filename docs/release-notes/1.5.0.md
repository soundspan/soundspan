# soundspan 1.5.0

## Share links

You can now share tracks, albums, and playlists with anyone — no account required. Grab a shareable link from the ··· menu on any track or from the action bar on an album page.

Shared links open a dedicated listening page that looks and feels like the real player: album art on the left, the full queue on the right, and the mini-player bar at the bottom. The page starts playing automatically when it loads, and the cover art and track info update as songs advance through the queue.

A few things you can control when generating a link:
- **Expiry date** — the link stops working after a chosen date
- **Max plays** — the link deactivates after it's been opened N times

Play-count limits now work correctly for albums and playlists. Previously each individual track stream counted as a play, meaning a 10-track album could burn through 10 plays in a single listen. Plays now count once per visit, not once per track.

## Downloading shared content

Share pages include download buttons for every track in the queue. The "Download All" button packages everything into a single ZIP file instead of triggering a separate browser download for each track.

Playlist share pages also offer JSON and M3U export if you want to import the playlist somewhere else.

## Managing links

The share link modal shows all active links for the current track or album and lets you revoke any of them on the spot.

## Other improvements

- Search queries made up entirely of very common words (like "the" or "a") now return partial matches instead of an empty results page.
- The health endpoint exposes version, uptime, and per-service latency for easier monitoring.
- Backend test coverage increased from ~80% to ~83%.
- Various internal reliability fixes around file path safety, partial archive handling, and test infrastructure.
